# 设计素材自动化生产工具 PRD

**版本** v0.1 · **状态** PoC 阶段  
**整理自** 产品讨论记录

---

## 1. 背景与目标

设计团队日常需要大量生产素材图，现有流程依赖 Photoshop 手动操作，效率低、重复劳动多。

**目标：** 通过 AI 对话 + penpot 模板库的组合，实现素材的半自动化批量生产，将重复性工作交给程序完成，设计师只需处理创意和二次修改。

**第一阶段目标（PoC）：** 跑通单条需求的完整链路，验证技术可行性。

---

## 2. 用户角色

| 角色 | 职责 |
|------|------|
| 设计师 | 在 penpot 维护模板库，按命名规范标注 slot，必要时对成品进行二次修改 |
| 运营 / 使用者 | 上传需求表格，通过对话选择模板，触发合成，导出成品 |
| 技术 | 维护本地服务，管理产品图库，维护系统运行 |

---

## 3. 整体架构

```
需求表格（Excel/CSV）
        ↓
AI 解析 → 结构化数据（产品名、文案、价格等）
        ↓
本地服务 → 匹配产品图库
        ↓
penpot API → 拉取模板结构 + 写入图片和文案
        ↓
导出 PNG → 可选切成九宫格
        ↓
输出成品 / 回流 penpot 供二次编辑
```

## 4. 界面设计

### 4.1 整体布局

三栏式布局：

```
┌─────────────┬──────────────────┬─────────────┐
│  左：模板库  │   中：合成预览   │ 右：AI对话  │
│  220px      │   自适应         │  280px      │
└─────────────┴──────────────────┴─────────────┘
```

### 4.2 左栏 — 模板库

- 从 penpot API 拉取模板缩略图，展示真实设计预览
- 支持按模板类型筛选：全部 / 4宫格 / 6宫格 / 9宫格 / 单品
- 点击模板卡片即选中，高亮显示当前选中状态
- 设计师更新 penpot 模板后，刷新即可同步最新缩略图

### 4.3 中栏 — 合成预览

- 实时展示当前合成状态
- 每个格子用颜色区分类型：蓝色=海报、绿色=产品、虚线=空白
- 展示进度步骤（解析中 / 匹配图片 / 合成中 / 完成）

### 4.4 右栏 — AI 对话

- 支持上传表格文件（Excel / CSV）
- AI 解析后以消息形式反馈结果和推荐模板
- AI 回复中提供快捷操作按钮（不需要手动打字）：
  - 导出 PNG
  - 切成九宫格
  - 在 penpot 修改
  - 换其他模板
- 输入框支持自然语言修改指令，例如：「把第3格换成产品E」

## 5. 核心功能

### 5.1 表格解析

- 支持格式：`.xlsx`、`.csv`
- AI 自动识别字段，无需固定列名
- 解析出结构化数据：产品名、文案、价格、规格、促销标签等
- 根据产品数量自动推荐模板类型（4 个产品 → 推荐 4宫格）
- **PoC 阶段：单条需求，即表格只有一行数据**

### 5.2 模板库管理

- 模板全部存储在 penpot，设计师自行维护
- 程序通过 penpot API 拉取模板列表和缩略图
- 模板按命名规范区分类型（见第 6 节）

### 5.3 产品图匹配

- 本地 Python 服务管理产品图库
- 根据产品名称精确匹配对应图片
- 匹配失败时在对话中提示用户手动指定

### 5.4 图片合成

- 通过 penpot API 写入产品图和文案到对应 slot
- 写入逻辑：按图层名称查找 slot，替换内容
- 文字样式（字体、颜色、大小）完全由 penpot 模板控制，程序只写内容
- 海报图 slot 保留，**第一阶段由人工在 penpot 填入**

### 5.5 导出

- 导出完整 PNG 成品
- 可选：自动切成九宫格（3×3），输出 9 张独立图片，按顺序命名
- 可选：保留 penpot 可编辑版本供二次修改

### 5.6 对话修改

- 修改本质是「用新参数重新生成」，不是真正修改
- 系统维护一个当前状态 JSON，每次修改只更新对应字段，其余不变
- 支持的修改指令：
  - 更换某格产品图
  - 修改文案
  - 调整字体大小
  - 换模板重新生成

## 6. 模板命名规范

### 6.1 核心原则

> **需要被程序替换的图层才命名，其余图层（背景、装饰、固定 logo 等）无需命名，设计师自由发挥。**

### 6.2 命名格式

所有需要程序处理的图层，统一使用 `slot/` 前缀：

```
slot/{类型}_{编号}/{字段}
```

### 6.3 各模板 slot 定义

**通用 slot（所有模板共用）**

| 图层名称 | 说明 | 是否必填 |
|---------|------|---------|
| `slot/poster` | 海报图位置，第一阶段人工填入 | 可选 |
| `slot/blank` | 空白格，程序跳过不处理 | 可选 |
| `slot/title` | 整体标题文案 | 可选 |

**产品格子 slot（每个产品格子一组）**

| 图层名称 | 说明 | 是否必填 |
|---------|------|---------|
| `slot/product_1/image` | 第1格产品图 | 必填 |
| `slot/product_1/name` | 第1格产品名称 | 必填 |
| `slot/product_1/price` | 第1格价格 | 可选 |
| `slot/product_1/tag` | 第1格促销标签（新品、热销等） | 可选 |
| `slot/product_1/spec` | 第1格规格说明 | 可选 |

以此类推，`product_2`、`product_3`……

**模板类型对应 slot 数量**

| 模板类型 | 产品 slot 数量 | 备注 |
|---------|--------------|------|
| 4宫格 | 4 个 | 无海报格 |
| 6宫格 | 4 个 | 1个海报 + 1个空白 + 4个产品 |
| 9宫格 | 7 个 | 1个海报 + 1个空白 + 7个产品 |
| 单品×N | N 个 | 视模板而定 |

### 6.4 可选字段处理规则

- 需求表格中该字段有值 → 写入对应 slot
- 需求表格中该字段为空 → 隐藏该图层，不影响其他 slot

---

## 7. 产品图库规范

- 统一存放在本地固定目录，例如 `/product-library/`
- 按产品名称命名文件，例如 `产品A.png`、`产品B.jpg`
- 建议使用透明背景 PNG，保证合成质量
- 图片分辨率建议不低于 800×800px

## 8. 技术栈推荐

### 8.1 总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端界面 | React + Vite | 三栏式 Web 界面 |
| AI 对话 | Claude API（claude-sonnet-4-5） | 表格解析、意图理解、修改指令 |
| 本地服务 | Python + FastAPI | 读取产品图库、驱动合成流程 |
| 模板管理 | penpot REST API | 拉取模板列表、缩略图、写入内容 |
| 图片处理 | Python Pillow | 九宫格切图、图片后处理 |
| 表格解析 | Python pandas + openpyxl | 读取 Excel / CSV |

### 8.2 本地服务接口（FastAPI）

```
GET  /templates          → 从 penpot 拉取模板列表和缩略图
POST /compose            → 触发合成流程
GET  /compose/{id}       → 查询合成状态
POST /export/png         → 导出完整 PNG
POST /export/grid        → 导出九宫格（切成9张）
```

### 8.3 合成状态 JSON 结构

```json
{
  "template_id": "penpot_frame_id",
  "template_type": "grid_6",
  "slots": {
    "poster": null,
    "product_1": {
      "image": "/product-library/产品A.png",
      "name": "产品A",
      "price": "¥99",
      "tag": "新品",
      "spec": null
    },
    "product_2": {
      "image": "/product-library/产品B.png",
      "name": "产品B",
      "price": "¥129",
      "tag": null,
      "spec": null
    }
  }
}
```

### 8.4 penpot API 关键调用

```
GET  /v1/files/{file_key}                    → 获取文件结构和图层信息
GET  /v1/files/{file_key}/nodes              → 获取指定节点详情
GET  /v1/images/{file_key}                   → 导出指定 frame 为图片（缩略图）
POST /v1/files/{file_key}/images             → 上传图片到 penpot
PATCH /v1/files/{file_key}                   → 写入文案和图片内容
```

## 9. PoC 验证范围

第一阶段只需跑通以下最小闭环：

- [ ] penpot API 连通，能拉取模板列表和缩略图
- [ ] 上传一张单条需求表格，AI 正确解析字段
- [ ] 本地服务匹配到对应产品图
- [ ] 通过 penpot API 写入产品图和文案
- [ ] 导出 PNG 预览图
- [ ] 切成九宫格输出 9 张图片

**PoC 不包含：**

- 多条需求批量处理
- 海报图自动匹配
- 复杂修改指令
- 用户权限管理
